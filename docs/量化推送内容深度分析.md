## 🔍 问题 1：Alpha 指标详解

### 1.1 Alpha 的定义

**Alpha（α）** 是金融学中的经典指标，表示**个股相对于市场基准的超额收益**。

在本系统中：
```
Alpha = 个股涨跌幅 - 大盘涨跌幅
```

### 1.2 代码实现位置

#### （1）Alpha 计算逻辑
**文件**：[`stock_monitor/core/workers/quant_worker.py`](file://d:/code/stock/stock_monitor/core/workers/quant_worker.py#L723-L724)

```python
# 第 723-724 行
benchmark_pct = self.engine.get_market_relative_strength()
alpha = ps["p_info"].get("pct", 0.0) - benchmark_pct
```

**解释**：
- `ps["p_info"].get("pct", 0.0)`：获取个股的涨跌幅（%）
- `benchmark_pct`：获取大盘（上证指数）的涨跌幅（%）
- `alpha`：两者之差，即个股跑赢/跑输大盘的幅度

#### （2）大盘基准获取
**文件**：[`stock_monitor/core/engine/quant_engine.py`](file://d:/code/stock/stock_monitor/core/engine/quant_engine.py#L577-L597)

```python
def get_market_relative_strength(self) -> float:
    """获取个股相对于大盘的强弱。当前取上证指数 (sh000001) 同比涨跌幅。"""
    try:
        idx_symbol = "000001"
        idx_market = 1  # SH

        # 检查基准缓存 (1分钟有效期)
        import time

        now_ts = time.time()
        if "benchmark" in self._market_benchmark_cache:
            cache_val, ts = self._market_benchmark_cache["benchmark"]
            if now_ts - ts < 60:
                return cache_val

        idx_info = self.get_latest_price_info(idx_symbol, idx_market)
        idx_pct = idx_info.get("pct", 0.0)
        self._market_benchmark_cache["benchmark"] = (idx_pct, now_ts)
        return idx_pct
    except Exception:
        return 0.0
```

**关键点**：
- **基准指数**：上证指数（sh000001）
- **缓存机制**：1 分钟有效期，避免频繁请求
- **异常处理**：失败时返回 0.0（中性基准）

### 1.3 Alpha 的实际意义

| Alpha 值 | 含义 | 投资建议 |
|----------|------|---------|
| **Alpha > 0** | 个股跑赢大盘 | ✅ 相对强势，值得关注 |
| **Alpha = 0** | 与大盘同步 | ➖ 跟随市场节奏 |
| **Alpha < 0** | 个股跑输大盘 | ⚠️ 相对弱势，需谨慎 |

**示例场景**：
```
大盘涨跌：+0.5%
个股涨跌：+1.75%
Alpha = 1.75% - 0.5% = +1.25%  ← 跑赢大盘 1.25%
```

### 1.4 优化建议

#### 问题：大盘数据获取失败时 Alpha 失真
如果 `get_market_relative_strength()` 返回 0.0（异常默认值），而实际大盘下跌 -2%，则：
```
个股涨跌：-1%
Alpha = -1% - 0% = -1%  ← 错误！实际应为 +1%（跑赢大盘）
```

#### 解决方案：增加数据有效性校验

```python
def get_market_relative_strength(self) -> tuple[float, bool]:
    """
    获取大盘基准涨跌幅

    Returns:
        (涨跌幅, 是否有效)
    """
    try:
        idx_symbol = "000001"
        idx_market = 1

        now_ts = time.time()
        if "benchmark" in self._market_benchmark_cache:
            cache_val, ts, is_valid = self._market_benchmark_cache["benchmark"]
            if now_ts - ts < 60:
                return cache_val, is_valid

        idx_info = self.get_latest_price_info(idx_symbol, idx_market)

        # 验证数据有效性
        is_valid = bool(idx_info and idx_info.get("price", 0) > 0)
        idx_pct = idx_info.get("pct", 0.0) if is_valid else 0.0

        self._market_benchmark_cache["benchmark"] = (idx_pct, now_ts, is_valid)
        return idx_pct, is_valid
    except Exception:
        return 0.0, False
```

然后在计算 Alpha 时：
```python
benchmark_pct, is_valid = self.engine.get_market_relative_strength()
if is_valid:
    alpha = ps["p_info"].get("pct", 0.0) - benchmark_pct
else:
    alpha = ps["p_info"].get("pct", 0.0)  # 大盘数据无效时，直接用个股涨跌幅
    app_logger.warning(f"大盘数据获取失败，Alpha 计算可能不准确")
```

## 🔍 问题 2：股价显示为 0.00 的原因分析

### 2.1 数据流追踪

#### （1）价格获取入口
**文件**：[`stock_monitor/core/engine/quant_engine.py`](file://d:/code/stock/stock_monitor/core/engine/quant_engine.py#L744-L755)

```python
def get_latest_price_info(self, symbol: str, market: int = None) -> dict:
    """获取最新价格和涨跌幅（从日线K线计算）"""
    try:
        df = self.fetch_bars(symbol, market, category=9, offset=3)
        if df.empty or len(df) < 2:
            return {}  # ← 空 DataFrame 或数据不足时返回空字典
        now = df.iloc[-1]["close"]
        prev = df.iloc[-2]["close"]
        pct = ((now - prev) / prev * 100) if prev != 0 else 0
        return {"price": now, "pct": pct}
    except Exception:
        return {}  # ← 异常时也返回空字典
```

**可能导致返回 `{}` 的情况**：
1. **网络超时**：MootDX 客户端连接失败
2. **数据源异常**：TDX 服务器无响应或返回空数据
3. **股票代码错误**：无效的 symbol 格式
4. **缓存缺失**：首次请求且网络失败
5. **日期解析错误**：K 线数据日期格式异常

#### （2）价格信息传递
**文件**：[`stock_monitor/core/workers/quant_worker.py`](file://d:/code/stock/stock_monitor/core/workers/quant_worker.py#L441)

```python

```

如果 `get_latest_price_info()` 返回 `{}`，则 `p_info = {}`

#### （3）Notifier 中的默认值处理
**文件**：[`stock_monitor/services/notifier.py`](file://d:/code/stock/stock_monitor/services/notifier.py#L187-L188)


**关键问题**：

- 导致推送中显示：`实时股价: 0.00`

### 2.2 根本原因总结

| 原因分类 | 具体场景 | 概率 |
|---------|---------|------|
| **网络问题** | MootDX 连接超时、TDX 服务器繁忙 | ⭐⭐⭐ 高 |
| **数据不足** | 新股上市、停牌股票 K 线数据少于 2 条 | ⭐⭐ 中 |
| **代码错误** | symbol 格式错误（如缺少市场前缀） | ⭐ 低 |
| **缓存失效** | 缓存过期后网络请求失败 | ⭐⭐ 中 |
| **异常捕获** | `fetch_bars` 内部异常被静默处理 | ⭐⭐ 中 |

### 2.3 优化方案

#### 方案 1：增强错误日志（推荐立即实施）

修改 `get_latest_price_info` 方法，记录详细错误信息：

```python
def get_latest_price_info(self, symbol: str, market: int = None) -> dict:
    """获取最新价格和涨跌幅（从日线K线计算）"""
    try:
        df = self.fetch_bars(symbol, market, category=9, offset=3)

        # 详细诊断日志
        if df.empty:
            app_logger.warning(f"[价格获取] {symbol} 返回空 DataFrame，可能原因：网络超时/数据源异常")
            return {}

        if len(df) < 2:
            app_logger.warning(f"[价格获取] {symbol} 数据不足（仅{len(df)}条），无法计算涨跌幅")
            if not df.empty:
                # 至少返回当前价格
                return {"price": df.iloc[-1]["close"], "pct": 0.0}
            return {}

        now = df.iloc[-1]["close"]
        prev = df.iloc[-2]["close"]

        if prev == 0:
            app_logger.warning(f"[价格获取] {symbol} 前一日收盘价为 0，数据异常")
            return {"price": now, "pct": 0.0}

        pct = ((now - prev) / prev * 100)
        return {"price": now, "pct": pct}

    except KeyError as e:
        app_logger.error(f"[价格获取] {symbol} 数据列缺失：{e}")
        return {}
    except Exception as e:
        app_logger.error(f"[价格获取] {symbol} 未知异常：{type(e).__name__}: {e}")
        return {}
```

#### 方案 2：多级回退策略（推荐中期实施）

```python
def get_latest_price_info_robust(self, symbol: str, market: int = None) -> dict:
    """
    健壮的价格获取方法（多级回退）

    回退策略：
    1. 尝试从日线 K 线获取（当前方法）
    2. 失败则尝试从实时行情接口获取
    3. 仍失败则从缓存中读取历史价格
    4. 最终失败返回 None（而非 0.0）
    """
    # 策略 1：日线 K 线
    try:
        result = self.get_latest_price_info(symbol, market)
        if result and result.get("price", 0) > 0:
            return result
    except Exception as e:
        app_logger.debug(f"[价格获取-策略1失败] {symbol}: {e}")

    # 策略 2：实时行情接口（如果可用）
    try:
        if hasattr(self.client, 'quotes'):
            q_df = self.client.quotes(symbol=[symbol])
            if q_df is not None and not q_df.empty:
                price = float(q_df.iloc[0]["price"])
                last_close = float(q_df.iloc[0]["last_close"])
                pct = ((price / last_close - 1) * 100) if last_close > 0 else 0.0
                return {"price": price, "pct": pct}
    except Exception as e:
        app_logger.debug(f"[价格获取-策略2失败] {symbol}: {e}")

    # 策略 3：从缓存读取（如果有）
    cached = self._bars_lru_cache.get((symbol, 9))
    if cached is not None and not cached.empty and len(cached) >= 2:
        try:
            now = cached.iloc[-1]["close"]
            prev = cached.iloc[-2]["close"]
            pct = ((now - prev) / prev * 100) if prev != 0 else 0.0
            app_logger.info(f"[价格获取-策略3成功] {symbol} 使用缓存数据")
            return {"price": now, "pct": pct}
        except Exception as e:
            app_logger.debug(f"[价格获取-策略3失败] {symbol}: {e}")

    # 所有策略失败
    app_logger.warning(f"[价格获取-全部失败] {symbol} 无法获取价格数据")
    return None  # 返回 None 而非 {}，便于上层判断
```

#### 方案 3：Notifier 层防御性处理（推荐立即实施）

修改 `dispatch_alert` 方法，优雅处理缺失的价格数据：

```python
@classmethod
def dispatch_alert(
    cls,
    config: dict,
    symbol: str,
    stock_name: str,
    signals: list[str],
    cycle_info: str = "",
    price_info: Optional[dict] = None,
) -> bool:
    """分发预警：优先使用企业应用通道，若未配置则回退到 Webhook 文字。"""
    if not signals:
        return False

    current_time = time.strftime("%H:%M:%S")

    # 【优化】优雅处理缺失的价格数据
    if price_info and price_info.get("price", 0) > 0:
        p = price_info.get("price", 0.0)
        pct = price_info.get("pct", 0.0)
        sign = "+" if pct >= 0 else ""
        price_display = f"{sign}{pct:.2f}%"
        price_detail = f"{current_time} 实时股价: ¥{p:.2f}"
    else:
        # 价格数据缺失时的降级显示
        price_display = "价格待更新"
        price_detail = f"{current_time} 实时股价: -- (数据获取中)"
        app_logger.debug(f"[推送降级] {symbol} 价格数据缺失，使用降级显示")

    title = f"🚨 异动: {stock_name} ({symbol}) {price_display}"

    # 信号行与历史回顾
    signal_rows = "\n".join([f"• {s}" for s in signals])
    desc_body = (
        f'<div class="gray">{price_detail}</div>\n'
        f'<div class="highlight">关键信号：</div>\n'
        f"{signal_rows}\n"
        f"--------------------\n"
        f"{cycle_info}"
    )

    # 1. 尝试企业应用
    if config.get("push_mode") == "app" or config.get("wecom_corpsecret"):
        res = cls.send_wecom_app_message(config, title, desc_body)
        if res:
            return True

    # 2. 回退到 Webhook
    webhook_url = config.get("wecom_webhook", "")
    body = f"{title}\n时间: {current_time}\n股价: {price_detail.replace(f'{current_time} ', '')}\n信号:\n{signal_rows}\n{cycle_info}"
    return cls.send_wecom_webhook_text(webhook_url, body)
```

| 价格缺失 | `贵州茅台 +0.00%` ❌ | `贵州茅台 价格待更新` ⚠️ |
| 详情显示 | `实时股价: 0.00` ❌ | `实时股价: -- (数据获取中)` ⚠️ |
