# 量化推送防抖动配置指南

## 概述

为避免量化信号（如 MACD 底背离、OBV 吸筹等）在连续扫描周期内被反复触发，导致企业微信等渠道收到大量重复推送，系统实现了智能的**防抖动与去重机制**。

## 核心机制

### 1. 信号冷却 (Cooldown)
- **原理**：为每个股票代码和信号类型设置冷却时间窗口
- **默认值**：30 分钟（1800 秒）
- **效果**：同一信号在冷却期内不会重复推送

### 2. 状态变更检测
- **原理**：记录上次推送时的评分，只有当评分发生显著变化时才重新推送
- **默认阈值**：评分变化 ≥ 2 分
- **示例**：
  - 首次推送：MACD 底背离（评分 +3）
  - 5 分钟后：评分仍为 +3 → **不推送**（冷却中）
  - 10 分钟后：评分变为 +5 → **推送**（评分显著变化）

### 3. 信号合并推送
- **原理**：将同一股票在同一扫描周期内触发的多个信号合并为一条精简消息
- **默认开启**：是
- **效果**：避免单只股票同时触发多个信号时刷屏

## 配置参数

在 `config.json` 或通过 UI 设置界面调整以下参数：

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `quant_alert_cooldown` | int | 1800 | 基础冷却时间（秒）。建议范围：600-3600（10分钟~1小时） |
| `quant_alert_score_threshold` | int | 2 | 评分变化阈值。超过此值才在冷却期内重新推送 |
| `quant_alert_merge_enabled` | bool | true | 是否启用信号合并推送 |

### 配置示例

```json
{
  "quant_alert_cooldown": 1800,
  "quant_alert_score_threshold": 2,
  "quant_alert_merge_enabled": true
}
```

## 高级调优建议

### 场景 1：高频交易者（希望更及时）
```json
{
  "quant_alert_cooldown": 600,      // 缩短至 10 分钟
  "quant_alert_score_threshold": 1,  // 降低阈值至 1 分

  "quant_alert_merge_enabled": true  // 合并推送
}
```

## 技术实现细节

### 缓存持久化
- **存储位置**：`.stock_monitor/cache/signal_cache.json`
- **格式**：JSON
- **自动清理**：超过 24 小时的缓存项会自动删除
- **重启恢复**：应用重启后会自动加载历史状态，避免重复推送

### 状态追踪结构
```json
{
  "sh600519::日线:MACD底背离": {
    "last_score": 3,
    "last_push_ts": 1712649300.123
  },
  "sh600519::30分钟:OBV碎步吸筹": {
    "last_score": 2,
    "last_push_ts": 1712649600.456
  }
}
```

## 常见问题

### Q1: 为什么我修改了配置但没有生效？
**A**: 请确保在 UI 设置界面点击"保存"按钮，或手动重启应用以重新加载配置。

### Q2: 如何清除所有信号的冷却状态？
**A**: 删除文件 `.stock_monitor/cache/signal_cache.json`，然后重启应用。

### Q3: 合并推送会不会遗漏重要信号？
**A**: 不会。合并推送会显示所有触发的信号名称和评分，并取最高分作为综合强度展示。

### Q4: 冷却时间是针对单个信号还是整只股票？
**A**: 针对**单个信号类型**。例如：
- `日线:MACD底背离` 和 `30分钟:OBV碎步吸筹` 是两个独立信号
- 它们各自有独立的冷却计时器

## 日志调试

如需查看防抖动逻辑的详细执行情况，可在日志中搜索关键词：
- `[symbol] {sig_name} 允许推送` - 信号通过检查
- `[symbol] {sig_name} 跳过` - 信号被拦截（冷却中）
- `发送合并推送` - 触发了合并推送模式
- `发送量化异动即时推送` - 触发了单独推送模式

示例日志：
```
[DEBUG] [sh600519] 日线:MACD底背离 允许推送: 新信号
[DEBUG] [sh600519] 30分钟:OBV碎步吸筹 跳过: 冷却中 (剩余1200s)
[INFO] 发送合并推送：sh600519 (2个信号)
```
