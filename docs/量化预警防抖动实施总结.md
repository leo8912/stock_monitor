# 量化推送防抖动优化 - 实施总结

## 📋 问题背景

**原问题**：量化信号（如 MACD 底背离）在连续扫描周期内被反复触发，导致企业微信等渠道收到大量重复或过于频繁的推送。

**影响**：
- 用户体验差：同一信号每分钟推送一次
- 信息过载：重要信号被淹没在重复消息中
- 资源浪费：频繁的网络请求和 API 调用

---

## ✅ 解决方案

### 1. 信号冷却机制 (Cooldown)
- **实现位置**：`QuantWorker._should_push_signal()`
- **核心逻辑**：
  ```python
  # 为每个 (symbol, sig_name) 维护独立冷却计时器
  if now - last_push_ts < cooldown:
      return False, "冷却中"
  ```
- **默认配置**：1800 秒（30 分钟）
- **可调整范围**：600-3600 秒（10分钟~1小时）

### 2. 状态变更检测
- **实现位置**：`QuantWorker._should_push_signal()`
- **核心逻辑**：
  ```python
  # 冷却期内，只有评分显著变化才重新推送
  score_diff = abs(current_score - last_score)
  if score_diff >= score_threshold:
      return True, "评分显著变化"
  ```
- **默认阈值**：2 分
- **示例场景**：
  - 首次推送：MACD 底背离（+3 分）
  - 5 分钟后：仍为 +3 分 → **不推送**
  - 10 分钟后：变为 +5 分 → **推送**（变化 ≥ 2）

### 3. 信号合并推送
- **实现位置**：`QuantWorker._merge_signals_for_symbol()`
- **核心逻辑**：
  ```python
  # 将同一股票的多个信号合并为一条消息
  if merge_enabled and len(pending_signals) > 1:
      merged = self._merge_signals_for_symbol(...)
      dispatch_alert(merged)
  ```
- **效果对比**：
  - **优化前**：3 条独立推送（刷屏）
  - **优化后**：1 条合并推送（精简）
    ```
    🚨贵州茅台 (sh600519) +1.2%
    信号组合: 日线:MACD底背离(+3), 30分钟:OBV吸筹(+2), 60分钟:RSRS走强(+4)
    综合强度: +4分
    ```

### 4. 配置化管理
- **新增参数**（`config.json`）：
  ```json
  {
    "quant_alert_cooldown": 1800,
    "quant_alert_score_threshold": 2,
    "quant_alert_merge_enabled": true
  }
  ```
- **修改文件**：`stock_monitor/config/manager.py`

### 5. 持久化缓存增强
- **缓存结构升级**：
  ```json
  // 旧格式
  {"sh600519::日线:MACD底背离": 1712649300.123}

  // 新格式（兼容旧格式）
  {"sh600519::日线:MACD底背离": {"last_score": 3, "last_push_ts": 1712649300.123}}
  ```
- **自动清理**：超过 24 小时的缓存项自动删除
- **重启恢复**：应用重启后加载历史状态，避免重复告警
