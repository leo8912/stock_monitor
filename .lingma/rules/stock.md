# 开发规则

## 代码规范

- 遵循 PEP 8 Python 代码风格指南
- 使用类型提示增强代码可读性
- 所有公共函数和类都需要有文档字符串
- 变量命名应具有描述性，避免使用单字母变量名

## 提交要求

- 每次提交应该只包含一个逻辑变更
- 提交信息需要清晰描述变更内容
- 所有测试必须通过后才能提交代码
- 提交前运行代码格式化工具

## 分支策略

- 主分支为 `main`，仅接受 Pull Request 合并
- 功能开发应在特性分支进行
- 分支命名规范：`feature/功能名称` 或 `bugfix/问题描述`
- **Git 操作由开发者手动执行，不要自动执行任何 Git 命令**

## 测试标准

- 新增功能必须包含单元测试
- 修改现有功能需要更新相关测试
- 测试覆盖率应保持在 80% 以上
- 集成测试需覆盖核心业务流程

## 部署流程

- CI/CD 自动化部署到测试环境
- 生产环境部署需要代码审查批准
- 部署后需要验证核心功能正常运行
- 所有 Git 操作均由开发者手动完成
