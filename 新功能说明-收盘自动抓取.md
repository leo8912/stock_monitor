## 一、暗盘资金数据显示功能

### 问题诊断

之前主界面暗盘列只显示"--"的原因是：**暗盘服务虽然已定义但从未被启动**。

### 解决方案

在 `MainWindowViewModel` 初始化时立即启动暗盘服务，并在每次数据刷新时将暗盘数据注入到股票行数据中。

### 实现细节

#### 1. 启动暗盘服务

文件：[`main_window_view_model.py`](file:///d:/code/stock/stock_monitor/ui/view_models/main_window_view_model.py:54)

```python
# Initialize Dark Trade Service and start it immediately
self._dark_trade_service = get_dark_trade_service()
if not self._dark_trade_service.isRunning():
    self._dark_trade_service.start_service()
    app_logger.info("[MainWindowViewModel] 暗盘资金服务已启动")
```

#### 2. 注入暗盘数据到股票行

文件：[`main_window_view_model.py`](file:///d:/code/stock/stock_monitor/ui/view_models/main_window_view_model.py:68)

```python
def _on_data_updated(self, stocks, all_failed):
    """Intercept local data updates to cache the latest data"""
    if not all_failed:
        # Inject dark trade data into stock rows
        self._inject_dark_trade_data(stocks)
        self._latest_stock_data = stocks
    self.stock_data_updated.emit(stocks, all_failed)

def _inject_dark_trade_data(self, stocks):
    """将暗盘资金数据注入到股票行数据中"""
    try:
        for stock in stocks:
            # 从暗盘服务获取该股票的净流入数据
            dark_flow = self._dark_trade_service.get_dark_flow(stock.code)
            if dark_flow is not None:
                stock.dark_flow_wan = dark_flow
                stock.dark_flow_valid = True
    except Exception as e:
        # 静默失败，不影响主数据显示
        app_logger.debug(f"注入暗盘数据失败: {e}")
```

## 技术细节

### 时间窗口设计

- **开始时间**: 15:05（收盘后5分钟，确保数据稳定）
- **结束时间**: 15:30（防止错过执行）
- **检查间隔**: 60秒（平衡精度和资源消耗）

### 防重复机制

使用日期标记 `_last_export_date`，确保同一天只执行一次：

```python
def _should_export_today(self) -> bool:
    now = datetime.now()

    # 周末不执行
    if now.weekday() >= 5:
        return False

    # 检查是否在当前时间窗口内
    t = now.time()
    if not (self._export_time_window_start <= t <= self._export_time_window_end):
        return False

    # 检查今天是否已经执行过
    today_str = now.strftime("%Y%m%d")
    if self._last_export_date == today_str:
        return False

    return True
```

### 异常处理

所有导出操作都包裹在try-except中，单个导出失败不影响其他导出：

```python
try:
    # 导出暗盘数据
    dark_file = export_dark_trade_excel(watchlist_codes=[])
    exported_files.append(str(dark_file))
except Exception as e:
    app_logger.error(f"[CloseExportScheduler] 暗盘数据导出失败: {e}")

try:
    # 导出自选股指标
    export_to_excel(...)
    exported_files.append(str(output_path))
except Exception as e:
    app_logger.error(f"[CloseExportScheduler] 自选股指标导出失败: {e}")
```

1. **仅在交易日执行**：周六、周日不会执行导出
2. **需要联网**：所有数据抓取都需要网络连接
3. **耗时较长**：完整导出可能需要30秒-2分钟，具体取决于股票数量和网络状况
4. **磁盘空间**：每个Excel文件约1-5MB，建议定期清理旧文件
