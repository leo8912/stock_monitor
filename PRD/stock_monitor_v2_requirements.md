# Stock Monitor V2 演进需求文档 (PRD)

**文档状态**: 草稿
**撰写日期**: 2026-03-27
**项目核心目标**: 从“基础实时看盘及大单监控表格”向“本地化智能量化大脑及自动预警推流中心”演进。

## 1. 核心业务痛点与现状分析
*   **同步不同源隐患**：当前系统使用了两套完全割裂的数据源，涨跌幅与现价使用 `easyquotation`（网络轮询 HTTP），明细数据使用 `mootdx`（TCP二进制流）。毫秒级的刷新误差会在高频行情阶段引发数据不对齐与涨跌幅矛盾。
*   **依赖桌面盯盘**：触发大单需要人眼持续盯着界面与颜色渐变。如果切到后台或离座，极易错过重点信号。
*   **基础的显示而不作分析**：系统缺乏“判定”与“总结”能力，不能根据积累的技术面或横切的财务面主动发掘并上报交易机会。
*   **置顶经常失效**：标准的 Qt `WindowStaysOnTopHint` 会被 Windows 系统的焦点争夺和 UI 瞬时阻塞破坏。

## 2. 核心功能演进需求 (New Features)

### 2.1 架构脱水与数据大一统
*   **需求描述**：全盘弃用 `easyquotation`，将“获取基础报价”的代码全部无缝替换为调用 `mootdx.quotes`。
*   **实现价值**：单库单连接，彻底消除价格跳动和明细弹出的时间差断层。

### 2.2 跨端推流报警与每日智能复盘 (Notification & Recap)
*   **自动复盘 (EOD Recap)**：每个交易日 15:05 分，系统自动汇总当日自选股异动（哪个大单净流入极长？哪个走出了顶背离？），写入 Markdown/纯文本文档。
*   **微信预警**：介入 `ServerChan` 或企业微信通道。脱离桌面界面，一旦盘中触发背离强记号或复盘生成完毕，立刻跨端推送至用户微信。

### 2.3 基于 pandas-ta 的量化技术共振报警
*   **需求描述**：接入量化运算模块，不占用复杂的界面绘制空间，改为“微信机器人消息”强推送。
*   **监控信号池**：
    *   **MACD 顶/底背离**（创新低但指标走高，完美极左侧抄底）。
    *   **主力碎步吸筹**：连日盘整但 OBV 量能潮或 VWAP 均价强托底。
    *   **布林带极度收敛爆发** (BBands Squeeze)。
    *   **关键反转 K 线形态**（早晨之星/吞没形态探测）。

### 2.4 长线筹码集中度监测架构
*   **需求描述**：利用 `mootdx.finance` 建立极低频的持久化“本地基准对账机制”。
*   **监控机制**：加入自选池瞬间，记录当天的“股东人数”存入本地微型数据库（如 TinyDB / JSON）。之后每日早间对比，一旦变动比例突破 5% (主力高度吸筹导致股东骤减，或主力大举派发导致激增)，进行强预警。

## 3. UI 与交互需求
*   **零界面干扰**：所有的量化运算、历史基准比对、定时大复盘，均须以**独立后台线程/进程 (Core Engine)** 托管，纯通过消息槽将预警结果广播到界面，保障 UI 不发生任何阻塞。
*   **修复强置顶**：不再依赖常规 Qt 层标志，改写使用 Windows 操作系统顶级穿透 API (`SetWindowPos` / `HWND_TOPMOST`)，保障界面置顶万无一失。
